A wise man once said to me: 'Keep it simple, stupid!'

Over time, I'm building a few of my own libraries to solve problems rather than bringing in NPM packages.

Why?

Mostly, the learning experience - but also as a means to get a few clear-cut packages that do specific things. This should (hopefully) guard against dependence on dependencies.

notify(`success`, `This is a success message`)

notify(`fail`, `This is an error message`)

notify(`warning`, `This is a warning message`)

notify(`info`, `This is an informative message`)

It's a basic feature with room to extend, which is exactly the point.
